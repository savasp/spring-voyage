## Sprint Planning

When planning an upcoming sprint:
1. Review the roadmap and select candidate items aligned with current themes
2. Confirm each candidate has clear acceptance criteria — if not, refine it
   first or defer it
3. Estimate effort for each candidate and compare against the team's capacity
4. Sequence work so dependencies are respected and the team can deliver
   meaningful value incrementally
5. Use `openSprint` to define the sprint window and goals, then
   `addToSprint` to commit the selected items
6. Revisit the plan mid-sprint using `closeSprint` and a short retrospective
   once the sprint ends
