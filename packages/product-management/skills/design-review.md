## Design Review

When reviewing a design proposal:
1. Confirm the user problem and success criteria the design is meant to solve
2. Check the design against these criteria:
   - Usability: clear affordances, low cognitive load, forgiving error states
   - Accessibility: meets the project's accessibility requirements
   - Consistency: reuses existing patterns and components from the design
     system; introduces new patterns only when justified
   - Implementability: the proposal can be built within the team's technical
     constraints
3. Leave specific, actionable feedback tied to parts of the design
4. Use `submitDesignReview` to record the decision (approve, request-changes,
   or comment) with a summary of the key points
5. If changes are requested, route feedback back to the designer and track
   the follow-up until the design is ready for handoff
